# Design Decisions Summary

This document logs the consensus-based decisions made during Kite design sessions. Detailed reasoning for major architectural shifts can be found in the `docs/adrs/` directory.

| Date | Decision | Context / Motivation | Associated ADR / Task |
|------|----------|----------------------|-----------------------|
| 2026-05-19 | Established Design Workflow | Need a structured way to run consensus-based design sessions without writing code directly. | N/A |
| 2026-05-19 | Virtual Markers for List Layout | Needed a way to render list bullets/numbers without violating the unified render box rule or causing clipping in the terminal grid. | ADR 001 |
| 2026-05-19 | Logical DOM Components for Lists | Introduced `ul`, `ol`, and `li` components. Decided to use `PaddingLeft` for terminal indentation and explicitly map `ListStyleType` as an inheritable style property. | Task: TSK-002 |
| 2026-05-19 | Table Layout and Fault Tolerance | Designed a two-pass algorithm for tables supporting ColSpan/RowSpan. Resolved to handle malformed tables strictly within the layout algorithm (like IFC anonymous blocks) to keep the engine sync phase pure. | ADR 002 |
| 2026-05-19 | Declarative UI API and DOM Adoption | Moved to a declarative API (`Box(Text("Hello"))`). Required modifying `dom.AppendChild` to implicitly adopt detached nodes from different documents. | ADR 003 |
| 2026-05-19 | Logical DOM Owns Element Styles | Moved ownership of `RawStyle` and `DefaultStyle` exclusively to the DOM elements. Render objects now proxy these calls. Maintains CSS cascade layering while avoiding state duplication. | Task: TSK-007 |
| 2026-05-19 | Table Grouping Elements | Decided to support `thead`, `tbody`, and `tfoot` logic in the table layout algorithm. This ensures perfect column alignment between sticky headers and data rows, and matches standard CSS behavior. Required adding anonymous row groups for fault tolerance. | ADR 002 |
| 2026-05-19 | Border Fluent API & Rendering | The `paint` engine's border rendering will be rewritten to respect per-edge widths, styles, and colors using defined Unicode glyph maps. To improve DevX, `style.Border` will act as an immutable fluent builder itself (e.g., `style.SingleBorder().Color(red)`), and we will create a type alias or property rename to avoid collision with `Optional[Border]`. | Pluralized struct fields (Edges, Styles, Colors) to allow singular fluent modifiers. Width changed to boolean Edges to reflect terminal constraints. | Task: TSK-009 |
| 2026-05-19 | Table Border Collapsing & Grid Painter | In a TUI, table cell borders must physically overlap by 1 cell and be drawn collectively to form correct Unicode intersections (e.g., `┼`, `├`). We will implement a specialized `paintTableGrid` pass triggered when `DisplayTable` is encountered in the paint engine, suppressing standard border painting for cells. | Task: TSK-010 |
| 2026-05-19 | Global Border Intersection Resolver | We replaced the `Table Grid Painter` with a global screen-space post-processing pass. The paint engine will flag all border cells with `FlagIsBorder`. At the end of the frame, a single $O(W \times H)$ pass resolves all intersecting borders (tables, sidebars, cards) into perfect Unicode junctions by checking neighbors. | Task: TSK-010 |
| 2026-05-19 | Table Layout Implicit Border Overlap | To support the Global Border Intersection Resolver, the Table Layout algorithms (`TableAlgorithm` and `TableRowAlgorithm`) must physically overlap adjacent cells by 1 coordinate space if they share a border. This ensures the paint engine plots their border glyphs on the exact same framebuffer cell, allowing the post-processor to correctly form cross-junctions (`┼`). | Merged into TSK-012 | Task: TSK-012 |
| 2026-05-19 | Table Layout Builder Pattern | To manage the complexity of two-pass table layout (grid sizing, spanning, grouping, and implicit border collapse), we will introduce a dedicated `TableFragmentBuilder` (or `GridSizingBuilder`) in the layout package. This builder will encapsulate the matrix math and state mutations, keeping `TableAlgorithm` focused purely on traversal logic. | Task: TSK-012 |
| 2026-05-19 | Layout Builder Standardization (Flex) | To enforce clean architecture across the Layout Engine, complex layout algorithms must separate coordination from state mutation via the Builder Pattern. Following Tables, the `FlexAlgorithm` will be refactored to use a `FlexLineBuilder` to handle item chunking, fraction distribution, and alignment math. | ADR 005 / TSK-013 |
| 2026-05-19 | Global Border Style Metadata | Replaced the boolean `FlagIsBorder` mask with an explicit `BorderStyle` enum on `paint.Cell`. This allows the border intersection resolver to handle mixed-style intersections using a "Heaviest Style Wins" precedence rule, removing the need for string sniffing. | ADR 006 / TSK-014 |
| 2026-05-20 | Input & TextArea Architecture | Input fields require scrollable text and cursor positioning without polluting the DOM with physical layout math. | ADR 007 / Tasks TSK-015 to TSK-018 |
| 2026-05-21 | UA Shadow Subtree for Replaced Elements | The "Replaced Element via Direct Casting" pattern (ADR-007) does not scale to compound widgets (checkbox, radio, select, slider) and duplicates styling guards. Replaced and compound elements now own a closed UA shadow subtree (engine-walked, dispatch-invisible, never focusable) via an internal `uaRoot`. Author code cannot reach it; `outer` retargets dispatch to the host. Supersedes ADR-007. | ADR 009 / TSK-018, TSK-024, TSK-025 |
| 2026-05-21 | Intrinsic Style Layer | UA-mandated properties (e.g., `display: inline-block`, `overflow: clip`, `white-space: pre-wrap`) need to be enforced declaratively above author styles in the cascade. Adds `IntrinsicStyle()` to `dom.Element` as the strongest cascade layer (origin `OriginUserAgent`); `DefaultStyle()` remains the weakest, author-overridable layer. | ADR 010 / TSK-022 |
| 2026-05-21 | Paint-Phase Overflow Clipping | The `style.Overflow` enum was unread by the paint engine; `Surface.Clip()` existed but had zero callers. Paint now applies content-box clipping during recursion for any fragment whose `OverflowX/Y` is `Hidden`, `Clip`, `Scroll`, or `Auto`. The fragment's own background and border paint unclipped at the parent's level; clipping applies to descendants only. Reuses existing `clippedSurface` machinery. | ADR 011 / TSK-027 |
| 2026-05-21 | Generic Scroll Offset on DOM Elements | Replaces the bespoke `scrollX`/`scrollY` machinery on `<input>` and `<textarea>` with a uniform browser-style scroll model: every `dom.Element` exposes `Scroll`, `ScrollTo`, `ScrollBy`; lazy `*scrollState` allocated only for scroll containers; paint clamps on read and translates child origins; default `Scrollable` wheel handler vended by the framework; new `event.EventScroll`. Adds `OverflowAuto` to the enum and a fluent `.Overflow(v)` shorthand on `style.Style` (keeps the X/Y split for asymmetric cases). | ADR 012 / TSK-028 |
| 2026-05-22 | Unified Text Control Base | Centralized duplicated caret positioning, scrolling math, and core event handling from `InputElement` and `TextAreaElement` into a shared `textControlBase` struct. Relies purely on the standard layout fragment bounds and `dom.Element.Scroll()` to handle viewports without creating specialized layout nodes, enabling future scrollbar UI support automatically. | ADR 013 / TSK-029 |
